# 언더그라운드맵 — 개발 계획 (v0.2 리빌드)

> 기준: `04_IA_사이트맵.md` · `05_화면기획서.md` · `design-system.html`.
> 목표: 기존 앱(앰버 팔레트·리스트 피드)을 **다크+주황 · 지도 중심 · 운영진 매장수급 모델**로 리빌드. 정적 HTML + Vanilla JS(ESM) + Supabase, 빌드 없음.

## 순차 개발 단계

| Phase | 산출물 | 파일 | 상태 |
|---|---|---|---|
| **0. Foundation** | 다크+주황 토큰·컴포넌트·지도핀·게이트·스플래시 CSS | `assets/css/app.css`, `index.html` | 진행 |
| **1. Data/Schema** | 매장중심 v0.2 데이터(이미지·시설·프로그램후보·핀상태) | `assets/js/data.js`, `supabase/schema.sql` | 예정 |
| **2. 진입/가입** | S0 스플래시(간판 스핀→네온), S1 회원가입(멀티스텝) | `assets/js/app.js` | 예정 |
| **3. 지도** | S2 다크맵 + 핀(●/🟠) + 지역전환 + 리스트시트 + 필터 | `app.js`, `app.css` | 예정 |
| **4. 매장상세** | S3 진행중(참여), S4 공간만(호스트) | `app.js` | 예정 |
| **5. 호스트/완료** | S5 프로그램 기획·대여신청, S6 신청완료 | `app.js` | 예정 |
| **6. 마이/이용권** | S7 내모임·나, S8 지역 이용권 | `app.js` | 예정 |
| **7. 폴리시/검증** | 광고 레일(신테마)·상태·반응형, 브라우저 골든패스 검증 | 전체 | 예정 |

## 원칙
- **골든패스 우선**: 스플래시→가입→지도→매장상세→신청까지 먼저 동작시키고 세부 확장.
- **mock 우선**: config.js 없이도 전체 흐름 데모(seed). Supabase 연결 시 실데이터.
- **각 Phase 후 브라우저 검증**(Playwright 스모크) → 회귀 방지.
- 지도는 **스타일라이즈드 다크맵(옵션 A)** 으로 구현(SDK 후순위).

## 데이터 델타(Phase 1에서 반영)
- `venues`: `images[]`, `facilities[]`, `program_candidates[]`, 좌표(mock `mx/my` %).
- `profiles`: `interests[]`, `bio`.
- 매장 상태 = 매장×활성모임 조인 파생(● 무 / 🟠 유).
- 매장 등록 = 운영진 시딩(앱 사용자 화면에 매장등록 폼 없음).
