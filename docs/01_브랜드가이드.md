# 언더그라운드맵 — 브랜드 가이드 (v0.1)

> 기준: `언더그라운드맵_확정기획_v2_2026-07-06.md` · 팀 일석삼조 (LBD 9기)
> 이 문서는 MVP 제작 시 카피·색·타이포·톤의 **단일 기준**이다.

---

## 1. 한 줄 정체성

> **"이 도시에 사는 사람만 아는 지도."**
> 관광객에게는 보이지 않고, 로컬 거주자만 들어올 수 있는 지도.
> 동네 매장의 유휴 시간·유휴 공간을, 로컬 호스트의 모임 무대로 바꾼다.

- **한글명:** 언더그라운드맵
- **영문명:** UNDERGROUND MAP
- **짧은 호칭:** 언맵 / UG MAP
- **카테고리 정의:** 시(市) 단위로 잠긴 **로컬 전용 커뮤니티 × 유휴공간 마켓플레이스**

---

## 2. 왜 '언더그라운드(지하)'인가 — 브랜드 코어

| 축 | 지상(관광 지도) | **지하(언더그라운드맵)** |
|---|---|---|
| 대상 | 누구나 | **그 도시 거주자만** |
| 진입 | 열려 있음 | **거주 인증으로 게이팅** |
| 콘텐츠 | 화려한 핫플·관광 큐레이션 | **로컬만 아는 유휴공간·동네 모임** |
| 감정 | 구경 | **소속·아지트·비밀** |

브랜드의 핵심 정서는 **"열림의 순간"** 이다. 인증을 통과하는 순간, 잠겨 있던 지도가 켜진다 — 이 전환(잠김 → 열림)을 UX·비주얼·카피 전반에서 반복적으로 연출한다.

---

## 3. 브랜드 퍼스낼리티 (5)

1. **로컬하다** — 우리 동네 언어로 말한다. 과장된 마케팅 톤 금지.
2. **은밀하다** — "아무나 못 본다"는 소속감. 요란하지 않게, 아는 사람끼리.
3. **따뜻하다** — 지하지만 차갑지 않다. 아지트의 등불 같은 온기.
4. **믿을 만하다** — 매장·호스트·참여자 모두의 몫을 지킨다(참가비 분배 원칙).
5. **가볍다** — 각 잡은 네트워킹이 아니라, 부담 없이 들르는 자리.

---

## 4. 톤 & 보이스

- **말투:** 담백한 반말~해요체 경계. 친근하되 들뜨지 않게.
- **하는 말:** "이 도시에 사는 사람만", "관광객은 모르는", "지도가 열렸어요", "오후 3시, 비어 있던 그 카페에서".
- **안 하는 말:** "역대급", "핫플", "인싸", 과장된 느낌표 남발, 관광·여행 프레임.
- **금지:** 거주 인증을 조롱/우회 조장하는 표현, 특정 지역 비하.

### 카피 예시
| 상황 | 카피 |
|---|---|
| 진입 게이트 | "이 지도는 **{도시}에 사는 사람**에게만 열립니다." |
| 인증 완료 | "{도시} 언더그라운드가 열렸어요." |
| 빈 피드 | "아직 조용한 동네네요. 첫 모임을 열어볼래요?" |
| 모임 카드 | "오후 3–5시 · 비어 있던 {매장}에서" |
| 참가비 안내 | "참가비의 일부는 공간을 내어준 매장에 돌아갑니다." |
| 호스트 유도 | "당신이 여는 자리가, 이 동네의 지도가 됩니다." |

## 5. 컬러 시스템

'지하 + 아지트 등불'을 표현하는 **다크 베이스 + 앰버 엠버(ember) 액센트**. 보조로 '지도 라인·열림 상태'를 나타내는 **시그널 틸(teal)**.

| 토큰 | HEX | 용도 |
|---|---|---|
| `--ug-bg` | `#0A0A0C` | 최하단 배경(지하) |
| `--ug-surface` | `#16161B` | 카드·시트 표면 |
| `--ug-surface-2` | `#1E1E25` | 표면 강조·입력창 |
| `--ug-line` | `#2A2A33` | 보더·구분선 |
| `--ug-ember` | `#FF8A3D` | **주 액센트**(CTA·강조·등불) |
| `--ug-ember-deep` | `#DB6B22` | 앰버 눌림/그라데이션 하단 |
| `--ug-signal` | `#35D2C5` | **보조**(열림·인증완료·지도 라인·상태 뱃지) |
| `--ug-text` | `#F4F4F3` | 본문 텍스트 |
| `--ug-muted` | `#9CA3AF` | 보조 텍스트·placeholder |
| `--ug-danger` | `#F26D6D` | 마감·경고 |

**글로우 규칙:** 주요 CTA·'열림' 순간에만 `box-shadow: 0 0 24px rgba(255,138,61,.35)` 수준의 은은한 발광. 남용 금지(은밀함 유지).

### 접근성
- 본문 텍스트는 `--ug-text` on `--ug-bg`로 대비 확보(AA 이상).
- 앰버/틸은 배경색으로 쓸 때 그 위 텍스트를 `--ug-bg`(어두운 글자)로 두어 대비 확보.

## 6. 타이포그래피

- **서체:** Pretendard (기존 산출물과 통일, 한글 가독성).
- **위계:**
  - Display(히어로): 28–34px / 800 / -0.02em, 필요 시 앰버 글로우.
  - H1 화면 제목: 20–24px / 700.
  - H2 섹션: 16–18px / 700.
  - Body: 15px / 400–500 / line-height 1.6.
  - Caption·메타: 12–13px / `--ug-muted`.
- **한글 줄바꿈:** `word-break: keep-all` (어절 단위).
- **숫자·시간(유휴시간·정원·참가비):** 살짝 강조(700), 필요 시 `--ug-signal`.

## 7. 로고 / 심볼

- **컨셉:** **키홀 핀(keyhole pin)** — 지도 핀의 안쪽 점이 열쇠구멍 모양. "잠긴 로컬 지도"를 한 형태로 표현.
- **상태 변형:**
  - 잠김: 아웃라인 + 회색(`--ug-muted`).
  - 열림(인증 후): 채워짐 + 앰버 글로우.
- **모노그램:** `UG` (앰버). 파비콘·앱 아이콘용.
- MVP 구현: 인라인 SVG(외부 이미지 의존 최소화, 배포 단순화). 아래 마크업을 사용.

## 8. UI 원칙 (MVP 레이아웃과 직결)

1. **앱 in 브라우저.** 화면은 **가운데 한 줄(핸드폰 너비, 최대 ~480px)** 로만 구성. 좌우는 비운다.
2. **좌우 여백 = 광고 레일.** 데스크톱에서 비는 좌우 공간에 광고 슬롯을 둔다. 모바일에서는 광고 레일 숨김(중앙 컬럼만).
3. **하단 탭바 + 상단 앱바** 로 앱 감을 준다(스퀘어테이블식).
4. **잠김 → 열림** 전환을 진입 플로우의 핵심 연출로.
5. 다크 유지. 밝은 배경 화면 만들지 않기.

> 광고·레이아웃의 구체 규격은 `03_개발사양.md §광고 레일`과 `index.html` 참조.
