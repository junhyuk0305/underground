# 언더그라운드맵 — 기능 정의 (v0.2)

> 기준: `언더그라운드맵_확정기획_v2_2026-07-06.md`
> 3자(매장·호스트·참여자) 구조를 그대로 기능으로 옮긴다. 이 문서는 **무엇을 만드는가**를 정의하고, **어떻게 만드는가**는 `03_개발사양.md`가 맡는다.
>
> **v0.2 변경점**
> 1. **게이팅 단위 세분화** — '시 단위' → **지역(region) 단위**로 일반화. **서울은 자치구(구) 단위**, 그 외 지역은 **시 단위**로 잠근다.
> 2. **지역 이용권(크로스-리전 패스)** 추가 — 홈 지역이 아닌 다른 지역에 참여하려면 유료 이용권을 받는다. 새 수익 라인이자 게이팅의 유일한 예외.

## 1. 3자 구조 → 역할(Role) 매핑

| 주체 | 앱 내 역할 | 핵심 행동 |
|---|---|---|
| 로컬 거주자 | `resident` (기본) | 거주 인증 → **홈 지역** 지도 열람 → 모임 신청·참가비 결제 |
| 방문자(이용권 소지자) | `visitor` (상태) | **이용권으로** 다른 지역 열람·모임 참여 (호스팅·매장등록 불가) |
| 호스트(매개자) | `host` | 프로그램 개설 → **홈 지역** 매장(공간)·시간 선택 → 모임 오픈 |
| 매장(공간 공급자) | `venue_owner` | **홈 지역**에 매장 등록 → 유휴시간·수용인원·이용시간 등록 |
| 플랫폼(일석삼조) | `admin` | 인증 승인·정산·시딩(초기 콘텐츠)·이용권 발행 |

> 한 계정이 여러 역할을 가질 수 있다(거주자이면서 호스트). 역할은 프로필의 플래그로 관리.
> `visitor`는 역할 플래그가 아니라 **유효한 이용권 보유 상태**로 성립한다(별도 가입 없음).

## 1-1. 지역(Region) 모델 — 서울=구 / 그 외=시

- 게이팅의 기본 단위는 **지역(region)** 하나의 테이블로 통일하고, 광역별로 세분 레벨을 다르게 둔다.
  - **서울특별시** → **자치구 25개**가 각각 독립된 지역(`kind='gu'`). 예: `마포구`, `영등포구`, `강남구`.
  - **그 외(경기도 등)** → **시**가 하나의 지역(`kind='si'`). 예: `수원시`, `성남시`.
- 이유: 서울은 생활권이 구 단위로 촘촘히 나뉘고 인구밀도가 높아 **구 단위가 곧 '동네'**다. 지방은 시 단위가 실질 생활권이다. → **밀도(광고 상품화 임계)와 로컬성**을 동시에 만족.
- 선택 UI: 광역(`metro`)으로 그룹핑된 `<optgroup>` 하나로 노출(서울 그룹 = 25개 구, 경기도 그룹 = 시 목록).
- **홈 지역(home region)** = 거주 인증을 통과한 지역. 사용자의 기본 열람·공급·개설 범위.

## 2. 불변 원칙 (기능이 절대 어기면 안 되는 것)

1. **지역 단위 게이팅** — 인증되지 않은 사용자는 그 지역의 매장·모임을 **볼 수 없다.** 지역 = **서울은 구, 그 외는 시.** (프론트 숨김이 아니라 DB 레벨에서 차단 → `03_개발사양.md` RLS)
2. **이용권 예외** — 게이팅을 넘는 **유일한** 방법은 유효한 **지역 이용권**이다. 이용권 소지자는 대상 지역을 **열람·참여**할 수 있으나 **공급(매장등록)·개설(호스팅)은 홈 지역 전용**(로컬 공급 무결성 보존).
3. **참가비 매장 분배 필수** — 모임에는 참가비가 있고, 그 일부는 반드시 매장에 분배된다. 참가비/분배율 필드는 모임 데이터의 필수값.
4. **공간이 최우선 자원** — 매장 없는 모임은 존재할 수 없다(모임은 항상 매장에 종속).

## 3. 기능 목록 (Feature List)

### A. 진입 · 인증 (게이트)
- **A1. 온보딩** — 서비스 한 줄 소개 + "잠긴 지도" 연출.
- **A2. 로그인/가입** — Supabase Auth(이메일 매직링크 or 전화 OTP).
- **A3. 지역 선택** — 광역별 `<optgroup>`에서 내 지역 선택. **서울은 구(자치구), 그 외는 시**를 고른다.
- **A4. 거주 인증(간이)** — MVP: 지역 선택 + 자기신고 체크("이 지역에 거주합니다") → `resident_verified` + 홈 지역(`region_id`) 부여. *실주소·행정 인증은 후순위(§7).*
- **A5. 열림 연출** — 인증 완료 시 "{지역} 언더그라운드가 열렸어요" 전환. (예: "마포구 언더그라운드가 열렸어요")

### B. 지도 · 탐색 (거주자 + 방문자)
- **B1. 모임 피드** — **현재 보는 지역**의 열린 모임 카드 리스트(모집중 우선, 마감 흐리게).
- **B1-1. 지역 전환기** — 피드 상단에 내가 볼 수 있는 지역 칩(홈 + 이용권 보유 지역) + `＋ 다른 지역`(이용권 화면 진입). 방문자 열람 시 **방문자 모드 배너** 노출.
- **B2. 필터** — 시간대(예: 오후 3–5시), 카테고리, 정원 여유.
- **B3. 모임 상세** — 프로그램 소개, 호스트, 매장(공간)·주소, 유휴시간, 소요시간, 정원/잔여, 참가비, 분배 안내. **타 지역이면 방문 배지 표시.**
- **B4. 지도 뷰(선택)** — 매장 위치 핀(키홀 핀). MVP는 리스트 우선, 지도는 후순위 가능.

### G. 지역 이용권 (크로스-리전 패스) ★ v0.2 신규
- **G1. 이용권 스토어** — 홈 지역이 아닌 다른 지역 목록 + 이용권 상품 선택(정기 30/90일, 단기 방문권 7일) → 발급.
- **G2. 내 이용권** — 보유 이용권 목록(대상 지역, 종류, 남은 기간 D-day), "이 지역 보기" 바로가기.
- **G3. 게이팅 해제** — 유효 이용권이 있는 지역은 **피드 열람 + 모임 참여**가 열린다(호스팅·매장등록은 여전히 홈 지역 전용).
- **G4. (백로그) 호스트 초대권** — 호스트가 자기 모임에 **특정 방문자 초대권을 발행·판매**(플랫폼 수수료). MVP는 플랫폼 발행 판매만.

### C. 참가 (거주자)
- **C1. 신청** — 정원 확인 후 신청 → `registration` 생성(상태: 신청됨).
- **C2. 참가비 결제** — MVP: **현장결제/외부링크 안내 + 신청 저장**까지. (PG 연동은 후순위 §7)
- **C3. 내 신청 내역** — 마이페이지에서 예정/지난 모임 확인, 취소.

### D. 호스트되기 (매개자)
- **D1. 호스트 전환** — 프로필에 host 플래그.
- **D2. 모임 개설** — 프로그램명·설명·카테고리 + **매장 선택**(등록된 매장 중) + 날짜/시간(매장 유휴시간 내) + 정원 + 참가비 + **매장 분배율**.
- **D3. 신청자 관리** — 내 모임의 신청 명단 확인.

### E. 매장 등록 (공급자)
- **E1. 매장 등록** — 상호·주소·카테고리·수용인원.
- **E2. 유휴시간 등록** — 요일/시간대(예: 평일 15–17시), 1회 이용가능 시간 길이.
- **E3. 매장 대시보드(간이)** — 우리 매장에서 열린 모임·예상 유입 확인.

### F. 공통
- **F1. 마이페이지** — 프로필, 역할, 내 도시, 로그아웃.
- **F2. 빈 상태/오류** — 조용한 동네·인증 필요·네트워크 오류 카피(브랜드 가이드 §4).
- **F3. 광고 레일** — 데스크톱 좌우 여백 광고 슬롯(모바일 숨김). → `03_개발사양.md §광고 레일`.

## 4. 화면(Screen) 정의 — 중앙 컬럼 앱 셸

> 모든 화면은 **핸드폰 너비 중앙 컬럼** 안에서 전환된다(SPA 화면 스위칭).

| # | 화면 | 키 | 주 진입 | 핵심 요소 |
|---|---|---|---|---|
| S0 | 스플래시/온보딩 | `onboarding` | 최초 진입 | 로고, 태그라인, [시작하기] |
| S1 | 로그인/가입 | `auth` | S0 | 이메일/전화 입력, OTP |
| S2 | 지역 선택 + 거주 인증 | `verify` | S1 | **지역 `<optgroup>`(서울=구·그외=시)**, 거주 체크, [지도 열기] |
| S3 | 모임 피드(홈) | `feed` | 인증 후 기본 | 앱바(지역명), **지역 전환 칩**, 모임 카드, 필터, 탭바 |
| S4 | 모임 상세 | `meetup` | S3 카드 | 매장·시간·정원·참가비, **방문 배지**, [신청] |
| S5 | 신청 완료 | `joined` | S4 | 확인·현장결제 안내, [내 모임] |
| S6 | 호스트: 모임 개설 | `host-new` | 탭바/마이 | 폼(프로그램·매장·시간·참가비·분배). **홈 지역 매장만** |
| S7 | 매장: 등록 | `venue-new` | 마이 | 폼(상호·주소·수용·유휴시간). **홈 지역에 등록** |
| S8 | 마이페이지 | `me` | 탭바 | 프로필, **보유 이용권**, 내 신청, 역할 전환 |
| S9 | 지역 이용권 | `passes` | 마이 / 피드 `＋` | 내 이용권 목록 + 지역·상품 선택 → [이용권 받기] |

**탭바(하단 4탭):** 지도(S3) · 개설(S6) · 등록(S7) · 나(S8). *역할 없는 사용자는 개설/등록 진입 시 전환 유도. 이용권(S9)은 마이/피드에서 진입.*

## 5. 핵심 사용자 흐름 (User Flows)

**참여자 골든패스**
`온보딩 → 가입 → 지역선택·거주인증 → (열림 연출) → 피드 → 모임상세 → 신청 → 완료(현장결제 안내)`

**방문자(크로스-리전) 골든패스** ★ v0.2
`피드 → ＋다른 지역 → 지역·상품 선택 → 이용권 받기 → (그 지역 피드로 전환) → 모임상세 → 신청`

**호스트 골든패스**
`마이 → 호스트 전환 → 모임 개설(홈 지역 매장·시간·참가비·분배) → 오픈 → 신청자 확인`

**매장 골든패스**
`마이 → 매장 등록(홈 지역) → 유휴시간 등록 → (호스트가 그 매장·시간으로 모임 개설) → 유입 확인`

## 6. MVP 스코프 (v0.2 = 이번에 만드는 것)

**IN (구현)**
- A1–A5 인증(간이 자기신고), **지역 단위 게이팅(서울=구/그외=시, RLS)**
- B1·B1-1·B3 피드·**지역 전환기**·상세 (B2 필터·B4 지도는 여유 시)
- C1·C3 신청·내역 (C2 결제는 **안내+저장까지**)
- D2 모임 개설(홈 지역), E1·E2 매장 등록(홈 지역)
- **G1–G3 지역 이용권 발급·보유·게이팅 해제**(결제는 발급 시뮬레이션까지)
- F1·F2·F3 마이·빈상태·광고 레일

**OUT (후순위, §7)**
- 실주소/행정 거주 인증
- PG 실결제·정산 자동화 (참가비 분배 + **이용권 결제·수수료 정산**)
- **G4 호스트 초대권 발행·판매**
- 푸시 알림, 채팅, 리뷰/평점
- 지도 SDK 기반 위치 뷰(초기엔 리스트)

## 7. 기획 오픈이슈와의 연결

기획 v2 §7의 미해결 질문이 기능 결정에 미치는 영향:

| 오픈이슈 | MVP 처리 | 향후 |
|---|---|---|
| 호스트 수익 모델 | 분배율 필드만 확보(매장 우선) | 호스트 몫·정산 UI |
| 거주 인증 방식 | 자기신고 체크(+지역 세분: 서울 구/그외 시) | 주소·위치·행정 인증 |
| 플랫폼 수수료 | 필드 미노출 | 참가비 5% + **이용권 수수료** 정산 흐름 |
| 크로스-리전 수요 | **지역 이용권**으로 흡수(열람·참여) + 방문자 상태 | 초대권 2차마켓·기업/이주자 번들 |
| 매장 '수요없음' 75% | 포트폴리오(성과) 화면으로 설득 근거 축적 | 매장 대시보드 고도화 |
| 콜드스타트 | admin 시딩(매장1·호스트1·참여자n) | 지역별 확장 플레이북 |

> 데이터 모델은 위 향후 항목(분배율·수수료·정산·이용권)을 **필드로 미리 자리 잡아** 후속 확장을 쉽게 한다. 상세는 `03_개발사양.md`.
