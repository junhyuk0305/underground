# 언더그라운드맵 — 화면기획서 (v0.2)

> 화면별 상세 스펙. 상위 구조는 `04_IA_사이트맵.md`, 시각 언어는 `design-system.html`, 데이터/RLS는 `supabase/schema.sql`.
> 모든 화면은 **폰너비 중앙 컬럼(max 480px)** 안에서 전환된다. 데스크톱은 좌우 광고 레일(≥1100px, 인증 게이트 화면 제외).
>
> **작성 방식:** 진입·인증(S0·S1)·참여·이용권(S3·S6·S8)은 병렬 서브에이전트 초안을 IA 기준으로 정규화·통합, 지도(S2)·호스트 공급(S4·S5)·마이(S7)는 직접 작성.

## 표기 규칙
- **핀 상태**: ● 어두운 핀(공간만) · 🟠 주황 네온 핀(프로그램 진행중)
- **컴포넌트**는 `design-system.html`의 이름 참조(btn--primary, .mcard, .pass, .mpin, .vbanner, .hl 등).
- **라우트**는 IA(docs/04) 기준: `#/splash #/signup #/map #/venue/:id #/host-apply/:venueId #/joined #/my #/passes #/me`.
- 카피는 **실제 문구 예시**(다크·로컬·은밀 톤).

# S0 · 스플래시 / 진입 (`splash`)

**목적/역할** 앱의 첫 0~3초. "관광객은 모르는 지하 지도"라는 온도를 **불이 켜지는 연출** 하나로 각인 + 백그라운드에서 `auth.getSession()`으로 세션·거주인증을 **선(先) 판별**하는 라우팅 게이트(사용자에겐 애니메이션만, 판별은 은밀히). 연출: 어두운 화면에 반투명하게 떠 있던 **로컬 가게 간판(키홀핀)** 이 휘리릭 회전하다 멈추고, 전원이 들어온 듯 형광 주황 네온으로 **'언더그라운드 맵'** 점등.

**진입·이탈** 진입=앱 최초 로드/콜드스타트/PWA 재실행(진입 전용). 자동 이탈=애니메이션+세션판별 중 늦게 끝나는 쪽 완료 시 분기 전환. 수동 이탈=탭/건너뛰기→종료 프레임 점프 후 전환.

**접근 가드** 없음(모두 통과). 단 **세션 판별 전에는 전환하지 않음**. 게이트 구간 → 광고 레일 미노출.

**레이아웃(위→아래)** ① 앱바 없음, 전체 `.gate`형 최심부 캔버스(`--bg-deep #050505`) ② 중앙 간판/키홀핀(`.keymark` 60×76, 주황 드롭섀도) ③ 네온 워드마크 '언더그라운드 맵' ④ (점등 후)태그라인 1줄 ⑤ 우상단 '건너뛰기'(최초만), 하단 판별 지연 시 로딩 점 `● ● ●`.

**모션 타임라인 (총 ≈2.8s / 최초)**
| 구간 | 시간 | 동작 | 이징 |
|---|---|---|---|
| P0 진입 | 0.0–0.3s | 간판이 반투명 실루엣(opacity .18, 회색, blur 6px)으로 떠 있음 + 상단 앰버 글로우 약하게 | fade-in |
| P1 회전 | 0.3–1.3s | `rotateY` 0°→~900°(2.5바퀴) '휘리릭', 동시에 blur→0·opacity→1·회색→주황 라인 | `--ease` 감속 |
| P2 정착 | 1.3–1.6s | 살짝 오버슛 후 '틱' 스냅 정지, scale 1.04→1.0 | spring |
| P3 점등 | 1.6–2.4s | 형광등 점멸(opacity 0→1→.35→1 2회) 후 완전 점등, 주황+text-shadow 글로우 0→full, 키홀핀 드롭섀도 강화 | flicker step + glow ease |
| P4 안착 | 2.4–2.8s | 태그라인·건너뛰기 `.reveal`(rise) | rise .5s |
| 전환 | 2.8s~ | 세션 판별 완료 시 크로스페이드 | fade .25s |

- **최소 노출**: 판별이 빨라도 최소 1.6s(점등)까지 유지. 2.8s보다 늦으면 점등 상태 유지 + 로딩 점 후 완료 시 전환.

**구성 요소** `.gate` 배경(radial 앰버 글로우 `.gate::before`), `<use href="#keypin">`+`.keymark`, 워드마크 `.gate h2 .em`(점등 토글 `is-lit`), 태그라인 `.gate p`, 건너뛰기 `.btn--ghost .btn--sm`, 토큰 `--ease/--glow/--brand-strong`.

**콘텐츠·카피** 워드마크 "언더그라운드 맵" · 태그라인 "관광객은 모르는 이 동네 지도" / "불이 켜졌어요. 이 동네에 들어오세요." · 건너뛰기 "건너뛰기" · 판별 지연 시 "동네를 여는 중…".

**데이터** 읽기: `auth.getSession()`→세션 있으면 `profiles`(resident_verified·region_id) 1건. 쓰기: 없음(단 `localStorage.ug_seen_splash` 플래그). RLS: `profiles` 본인 행만. 사용자 노출 데이터 없음(연출 전용).

**상태** 로딩=애니메이션이 곧 인디케이터(초과 시 로딩 점) / 빈=비로그인(정상 분기) / 에러=세션조회 실패 시 **안전기본값=비로그인**으로 간주해 S1(조용한 실패) / 성공=분기 전환.

**인터랙션·전환(분기 핵심)**
- 세션 **없음** → `#/signup`(S1).
- 세션 **있음** & (프로필 없음 or `resident_verified=false`) → `#/signup`의 **거주 인증 스텝**.
- 세션 **있음** & `resident_verified=true` → `#/map`(S2, 홈 지역으로 바로).
- 탭/건너뛰기 → 종료 프레임 점프 → 판별 완료 즉시 전환.

**엣지** prefers-reduced-motion=회전·점멸 제거, 페이드로 최종 점등만(0.3s)+대기 1.0s / 재방문(`ug_seen_splash`)=짧은 0.8s / 저사양=blur 조기 제거·reduced 폴백 / 매직링크 콜백 딥링크=스플래시 스킵 / 탭 절전=타이머 일시정지 / 세션 만료=비로그인 분기.

**신모델·v0.2** 스플래시=세션 판별 로더 겸 게이팅 관문. 매장 공급은 admin 협의 등록이라 진입 단계에 사장님/매장 CTA 없음(공통 진입). 네온 주황="불이 켜진다"—지도 🟠핀과 은유 통일.

# S1 · 회원가입 / 로그인 (`signup`)

**목적/역할** 처음 온 사람을 **닉네임 → 관심사 → 지역·거주 인증 → 위치 동의** 짧은 온보딩으로 통과 → **'{지역} 언더그라운드가 열렸어요'** 열림 연출로 지도(S2) 진입. 로그인=이메일 매직링크(기존=로그인/신규=가입 자연 통합). 게이팅 시작점: 여기서 `resident_verified=true`+홈 `region_id` 부여돼야 그 지역 지도가 열린다.

**진입·이탈** 진입=S0(비로그인) 또는 매직링크 콜백 재진입. 이탈=온보딩 완료→열림 연출→`#/map`. 재진입 시 미완성 스텝부터 복원.

**접근 가드** `resident_verified=true`면 접근 시 `#/map`로 리다이렉트(중복 온보딩 방지). 매직링크 발송/콜백은 세션 없이 접근 가능. 광고 레일 미노출.

**레이아웃(위→아래)** — 스텝형(상단 진행 도트, 하단 고정 `.cta-dock`)
- **모드 A · 로그인/매직링크(콜백 전)**: ① 앱바(← + "시작하기") ② 히어로 카피 ③ `.field` 이메일 인풋 ④ `.cta-dock` [매직링크 받기] ⑤ 발송 후 `.hl` "메일함 확인" + [메일 앱 열기]/[재발송].
- **모드 B · 온보딩 폼(콜백 후)**: 스텝 도트(●●○○) ① **닉네임** `.field` ② **관심사** 멀티선택 칩 그리드 ③ **지역·거주 인증** `<select>` optgroup(서울=구/그외=시) + 거주 자기신고 체크 + `.hl` 안내 ④ **위치 동의** 설명 카드 + [위치 허용]/[나중에] ⑤ 각 스텝 하단 [다음]/마지막 [지도 열기].

**구성 요소** `.appbar`, `.field`+`.inp`(주황 포커스링), 관심사 `.chip[aria-pressed]`(**멀티선택=토글식**, 단일선택 JS와 구현 주의), 지역 native `<select>`+`<optgroup>`(스타일=.inp), 거주 체크 `.lrow`+check(온 시 `--brand-tint`), `.hl` 안내, `.btn--lg/--primary`·`--neutral`·`--ghost`, 스텝 도트, 성공=`.gate` 재사용(키홀핀 점등), `.toast`.

**콘텐츠·카피**
- 모드A 제목 "이 동네에 이메일로 조용히 들어오기" / 서브 "비밀번호 없이, 메일로 온 링크 한 번이면 돼요." / CTA "매직링크 받기" / 발송 후 "메일함을 확인하세요. **{이메일}** 로 입장 링크를 보냈어요. 링크를 누르면 이 동네가 열립니다."
- 스텝1 "표시 이름" / "동네에서 불릴 이름" / "실명 아니어도 돼요. 언제든 바꿀 수 있어요."
- 스텝2 "무엇에 끌리세요?" / "관심사에 맞는 이 동네 모임을 먼저 보여줄게요. (1~5개)" / 칩: 취미·공예 / 모각작 / 로컬 탐방 / 러닝·산책 / 사진·필름 / 책·글쓰기 / 음악·LP / 요리·베이킹 / 보드게임 / 커피·와인
- 스텝3 "어느 동네에 사세요?" / "서울은 구, 그 외 지역은 시 단위로 열려요." / 라벨 "거주 지역" / 체크 "네, 이 지역에 살고 있어요" / `.hl` "지금은 **자기신고**로 열어드려요. 신분증·주소 인증은 곧 추가돼요." (예: 마포구)
- 스텝4 "지금 있는 곳을 잠깐 볼게요" / "가까운 동네 모임을 위로 올려주고, 거주 확인을 도와요. 안 켜도 이용엔 문제없어요." / [위치 허용]·[나중에]
- 마지막 CTA "지도 열기" / 열림 연출 "**마포구** 언더그라운드가 열렸어요"(지역 동적) / 토스트 "입장 링크를 보냈어요." · "메일을 보내지 못했어요. 잠시 후 다시 시도해요."

**데이터** 읽기: `regions`(optgroup, 서울 kind='gu'/그외 'si'), 콜백 후 `auth.getUser()`/`profiles`. 쓰기: `auth.signInWithOtp({email, emailRedirectTo})`; `profiles` upsert(display_name, region_id, resident_verified, is_host/is_venue_owner=false); **`interests`(신규 필드, 부록C)**. RLS: `profiles` 본인만 insert/update, `regions` 전체 열람. **홈 지역=이후 모든 게이팅 기준(`my_home_region()`)**.

**상태** 로딩=발송/세션교환 로더 / 빈=`regions` 로드 실패 시 "지역을 불러오는 중…"+재시도 / 에러=이메일형식(인라인)·발송실패(토스트+30s 쿨다운 재발송)·**만료된 매직링크**(전용 카드+재발송)·프로필저장실패(값 보존) / 권한없음(위치 거부)=폴백 진행("위치 없이 계속했어요") / 성공=거주체크+지역확정+저장→열림 연출(≈1.4s)→`#/map`.

**인터랙션·전환** [매직링크]→발송→대기카드→(링크 클릭)→콜백 모드B 스텝1. 스텝 1→2→3→4(뒤로 값 보존, 필수 미충족 시 CTA 비활성). 위치 [허용/나중에]→다음. [지도 열기]→열림 연출 오버레이→`#/map` 크로스페이드.

**엣지** 이미 가입된 이메일=로그인 처리(인증완료면 온보딩 스킵) / 이탈 후 재진입=미완성 스텝 복원 / 닉네임 2~16자(중복 허용, 금칙어 후순위) / 관심사 최소1·최대5 / 지역·거주 필수(우회 불가) / 실거주와 다른 자기신고=MVP 허용(향후 검증) / 서울은 시 단독 선택 불가·구까지 / 매직링크 타 기기 열림 안내 / 재발송 쿨다운.

**신모델·v0.2** **관심사(interests) 필드 신규 필요**(부록C). 온보딩은 `resident`만 부여, is_host/is_venue_owner=false(호스트 전환은 어두운 핀→S5에서). 지역=서울 구/그외 시(optgroup). 거주=자기신고(MVP), 위치는 보조 신호. 열림 연출=브랜드 정체성(S0 점등과 통일).

# S2 · 메인 지도 (`map`) ★ 앱의 심장

**목적** '언더그라운드' 다크 지도 위에서 **이용 가능한 공간(가게)** 을 한눈에. 핀 색으로 "지금 놀 수 있는 곳(🟠)"과 "내가 열 수 있는 빈 공간(●)"을 구분.

**진입·이탈** 탭바 홈. 인증 후 기본. 핀 탭 → 매장 상세(S3/S4). 지역 ▾ → 지역 전환/S8.

**접근 가드** 로그인 + 거주 인증 필수. 미인증 → 게이트(빈 지도 금지). `viewRegion` 볼 권한 없으면 이용권 유도.

**레이아웃(위→아래)**
- **앱바**: keypin 로고 + **현재 지역명 "마포구 ▾"**(탭=지역 전환) + (우) 필터 아이콘.
- **방문자 배너**(조건): `viewRegion ≠ 홈`이면 `.vbanner` "방문자 모드 · 강남구를 이용권으로 보는 중 · D-6".
- **지역 전환칩 `.regionbar`**: [📍마포구(home,pressed)] [강남구·이용권 `.rchip--pass`] [＋ 다른 지역 `.rchip--add` → S8].
- **지도 캔버스**(주 영역): 다크 스타일 지도. 핀 `.mpin--live`(🟠 펄스) / `.mpin--dim`(● 어두운 점). 내 위치 점, 줌/팬.
- **하단 리스트 시트**(드래그업): 현재 지역 매장/프로그램 카드(`.mcard`), 정렬(가까운/임박/신규), 필터칩(관심사·시간·상태).
- **탭바**: 지도(on) · 내 모임 · 이용권 · 나.

**핀 탭 → 미니 프리뷰(바텀시트)** 매장 썸네일·이름·상태 뱃지·(🟠)프로그램 제목/시간·(●)"공간만 · 프로그램 열 수 있어요" → [자세히 보기] → S3(🟠)/S4(●).

**구성 요소** 지도+`.mpin`, `.regionbar`, `.vbanner`, 필터 `.chip`, 매장 카드 `.mcard`, 바텀시트 핸들, FAB(내 위치).

**콘텐츠·카피** 빈 상태 "이 동네는 아직 조용해요. 첫 불을 켜보실래요?" [빈 공간 둘러보기]. 필터 "진행중만 / 빈 공간만 / 오후 3–5시".

# S6 · 참여 신청 완료 (`joined` · `#/joined`)

**목적/역할** 신청이 저장됐다는 **완결 연출** + 현장결제·매장 분배 안내(MVP=PG 미연동, 신청 저장까지) + 다음 행동.

**진입·이탈** 진입=S3에서 신청 성공 직후(유일 정상). 이탈=[내 신청 보기]→나(S7a) 내 신청 · [지도로]→S2.

# S8 · 지역 이용권 (`passes` · `#/passes`)

**목적/역할** **보유 이용권 관리**(대상 지역·종류·D-day, "이 지역 보기") + **다른 지역 발급**(정기 30/90일·단기 7일) → 게이팅의 유일한 예외. 발급=결제 시뮬레이션(MVP), 발급 후 해당 지역 지도로 전환.

**진입·이탈** 진입=나(S7b) [보유 이용권]/S2 지역칩 `＋ 다른 지역`/S3 "이용권 필요" CTA. 이탈=발급 완료→**대상 지역 지도(S2)로 전환**(방문자 모드)·[이 지역 보기]→그 지역 지도·[←]→직전.

**접근 가드** 인증·거주 미완료→`#/signup` 인증 스텝. 발급 대상은 **홈 지역 제외**. `access_passes INSERT` RLS=`user_id=auth.uid() AND region_id ≠ my_home_region()`.

**레이아웃(위→아래)** ① 앱바([←]·"지역 이용권") ② **내 이용권**(라벨 "보유 중인 이용권" + `.pass` 리스트: 대상 지역·종류·D-day·[이 지역 보기]; 없으면 빈 상태) ③ 설명 배너(이용권 개념 한 줄) ④ **다른 지역 발급**(라벨 "다른 지역 열어보기": 지역 `<optgroup>` 셀렉트[홈·보유 제외/비활성] + 상품 카드 3종[`.pass` 라디오]) ⑤ 선택 요약 + sticky [이용권 받기].

**구성 요소** 보유 `.pass`(pk/pt/pm/pd)+[이 지역 보기] `.btn--sm.btn--soft`, 지역 `.inp` select+optgroup, 상품 `.pass` 라디오(선택 시 `--brand-line`+check), 발급 `.cta-dock`+`.btn--lg`, 빈 상태 `.gate` 톤/캡션, `.toast`.

**콘텐츠·카피** 타이틀 "지역 이용권" · 설명 "이용권이 있으면 홈이 아닌 지역도 열람하고 모임에 참여할 수 있어요. 단, 호스팅과 매장은 홈 지역에서만." · 보유 예 "정기 이용권 · 90일 / 강남구 언더그라운드 / 열람·참여 가능 / D-72 / [이 지역 보기]" · 빈 "아직 받은 이용권이 없어요. 다른 동네가 궁금하면 아래에서 열어보세요." · 지역 placeholder "지역 선택 (서울=구 · 그 외=시)" · 홈 힌트 "마포구는 홈 지역이라 이미 열려 있어요." · 상품: "정기 이용권 · 30일 — ₩9,900 — 한 달 동안 열람·참여" / "정기 이용권 · 90일 — ₩24,900 — 가장 인기 · 하루 277원꼴" / "단기 방문권 · 7일 — ₩4,900 — 짧게 다녀올 때" · 요약 "강남구 · 정기 90일 · ₩24,900" · CTA [이용권 받기] / 홈 disabled "홈 지역은 발급할 수 없어요" / 보유 "이미 이용권이 있어요"(soft) · 성공 토스트 "강남구 언더그라운드가 열렸어요.".

**데이터** 읽기: `access_passes`(내 것: region_id,kind,price,valid_from,valid_until,status; D-day=ceil((valid_until−now)/1d), status='active' AND valid_until>now만 유효) RLS SELECT 본인; `regions`(optgroup, 홈·보유 제외); `PASS_PRODUCTS`(클라 상수). 쓰기: `db.buyPass()`→`access_passes` INSERT(user_id,region_id,kind,price,issued_by='platform',valid_from=now,valid_until=now+days,status='active'), RLS `region_id ≠ my_home_region()`, 불변식 valid_until>valid_from·홈 발급 금지. 결제: PG 미연동→발급 시뮬레이션(즉시 active).

**상태** 로딩(스켈레톤) / 빈(보유 0→발급 유도) / 에러(발급 실패 토스트+롤백) / 권한없음(홈·보유 지역 선택→CTA disabled+힌트) / 성공(INSERT→`viewRegion`=발급 지역→S2 전환+토스트+방문자 배너).

**인터랙션·전환** [이 지역 보기]→`viewRegion` 변경→S2(방문자 배너). 지역 선택→상품 활성→선택→요약·CTA 금액 갱신. [이용권 받기]→시뮬레이션→성공 토스트 "{지역} 언더그라운드가 열렸어요."→S2(대상 지역). 실패→토스트+선택 유지.

**엣지** 홈 발급 시도(선택기 비활성+CTA disabled+RLS 이중 방어) / 중복 발급(→[이 지역 보기] 유도) / 만료(D-day≤0 "만료됨" 회색+[다시 받기], `can_view_region` 자동 제외) / 임박(D-3↓ "곧 만료 · 연장" 강조) / guest 7일(종류 뱃지·짧은 기간). 백로그 G4 호스트 초대권은 이 화면 밖.

**신모델·v0.2** 이용권=게이팅의 유일한 예외(열람·참여 O / 호스팅·매장 X, 경계 명시). 서울=구/그외=시 region 단위(optgroup=metro 그룹핑). 발급 후 즉시 그 지역 지도 전환("열림 연출" 이용권 버전). 수익=결제액(초기 전액 플랫폼, MVP 수수료 필드 미노출).
