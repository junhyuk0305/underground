# 언더그라운드맵 — IA · 사이트맵 (v0.2)

> 정보구조(Information Architecture)의 단일 기준. 개별 화면 상세는 `05_화면기획서.md`.
> 이 문서는 **화면 지도 · 네비게이션 · 라우팅/가드 · 전역 상태 · 매장 라이프사이클**을 정의한다.

---

## 0. 서비스 모델 (2026-07-06 확정 — 화면 설계의 근거)

> **운영진이 로컬 가게의 유휴 공간을 협의·등록 → 호스트가 그 공간에 문화 프로그램을 기획해 열고 → 참여자가 신청해 소비하는, 지역으로 잠긴 다크 지도.**

핵심 축(이전 문서와의 **델타 명시**):

1. **매장 수급 = 운영진(admin) 주도.** 운영진이 지방 로컬 가게와 공간 대여를 협의하고, 그 자리에서 **"이 공간에서 실행 가능한 프로그램 후보 + 보유 시설/환경"** 을 함께 확정해 매장으로 등록한다.
   → **호스트·사장님이 앱에서 직접 매장을 등록하지 않는다.** (`02_기능정의.md`의 E.매장등록은 **admin 백오피스**로 이동. 앱 사용자 화면에서는 제거.)
2. **매장은 지도에서 두 상태로 산다** (§5 라이프사이클):
   - **어두운 핀** = 공간만 대여가능, 아직 프로그램 없음(호스트 대기).
   - **주황 네온 핀(펄스)** = 호스트가 프로그램 기획·대여 매칭 완료, 프로그램 진행중.
3. **호스트 여정**: 어두운 핀 → 매장 상세(이미지·시설·프로그램 후보) → **프로그램 기획 + 공간 대여 신청** → 매칭 시 주황 핀 전환.
4. **참여자 여정**: 주황 핀 → 매장 상세(이미지·시설·진행 프로그램·호스트) → **참여 신청**. 참가비 일부는 **반드시 매장에 분배**(불변).
5. **게이팅**: 서울=자치구(구), 그 외=시. 거주 인증 안 되면 그 지역 지도가 **빈다**(RLS). 다른 지역은 **지역 이용권**으로만 열람·참여(방문자 상태). **공급·호스팅은 홈 지역 전용.**

## 1. 역할 · 상태 모델

| 주체 | 종류 | 성립 조건 | 앱에서 할 수 있는 것 |
|---|---|---|---|
| 참여자(resident) | 기본 역할 | 홈 지역 거주 인증 | 홈 지역 지도 열람 · 프로그램 참여 신청 |
| 방문자(visitor) | **상태**(역할 아님) | 대상 지역 **유효 이용권** 보유 | 그 지역 열람 · 참여 (호스팅·공급 불가) |
| 호스트(host) | 상태/플래그 | 홈 지역에서 대여 신청·매칭 | 어두운 매장에 프로그램 기획·대여 신청·운영 |
| 운영진(admin) | 백오피스 | — | 매장 섭외·등록(시설·프로그램후보)·승인·정산·이용권 발행 |

- 한 계정이 참여자↔호스트를 오간다(별도 가입 없음). **호스트는 "어두운 매장에 프로그램 열기"를 누르는 순간 진입**한다.
- 방문자는 가입 절차가 아니라 **이용권 보유 상태**로 성립·소멸(만료).

## 2. 사이트맵 (화면 목록)

```
언더그라운드맵 (앱 사용자)
│
├─ S0  스플래시/진입           splash        [간판 스핀→네온 점등]
├─ S1  회원가입/로그인          signup        [닉네임·관심사·거주인증·위치동의]
│
├─ [탭] 지도                    map (S2)      ← 앱의 홈/심장
│        ├─ S3 매장상세·진행중   venue-live    (주황 핀 → 참여)
│        │     └─ S6 신청완료    joined
│        └─ S4 매장상세·공간만   venue-open    (어두운 핀 → 호스트)
│              └─ S5 프로그램 기획·대여신청  host-apply
│
├─ [탭] 내 모임                 my (S7a)      [참여 신청 · 내 호스팅]
├─ [탭] 이용권                  passes (S8)   [보유 이용권 · 다른 지역 열기]
└─ [탭] 나                      me (S7b)      [프로필·관심사·설정·로그아웃]

운영진(admin) — 별도 백오피스(후순위, 앱 밖)
└─ A0 매장 큐레이션            admin/venues  [가게 섭외·시설·프로그램후보 등록·매칭 승인]
```

## 3. 네비게이션 구조

### 하단 탭바 (4탭, 인증 후 전역)
| 탭 | 화면 | 아이콘 | 역할 |
|---|---|---|---|
| **지도** | S2 `map` | map | 홈. 지역 지도·핀 탐색 |
| **내 모임** | S7a `my` | check/list | 내 참여 신청 + 내 호스팅 |
| **이용권** | S8 `passes` | pass | 보유 이용권 · 다른 지역 열기 |
| **나** | S7b `me` | user | 프로필·관심사·인증·설정 |

> 이전 v0.2 탭(개설·매장 등록)은 **제거**. 개설=지도의 어두운 핀에서 진입, 매장 등록=운영진 백오피스로 이동.

### 상단 앱바
- **지도(S2)**: 로고 + **현재 지역명 + ▾**(탭 시 지역 전환 시트). 우측 필터/검색(선택).
- **상세/폼(S3~S6)**: ← 뒤로 + 화면 제목. 상세는 스크롤 시 이미지→축소 앱바.
- **진입/게이트(S0~S1)**: 앱바 없음(풀블리드).

### 진입점(Entry points)
- 최초 실행 → S0 → (비로그인)S1 / (로그인·미인증)S1 거주인증 / (인증완료)S2.
- 매직링크 콜백 → 세션 복원 → S2(또는 미인증 시 S1 인증 단계).
- 딥링크(선택): 매장 상세/이용권.

## 4. 라우팅 & 가드

해시 라우팅(`app.js`). 기존 라우트에서 신모델로 조정:

| route | 화면 | 가드 |
|---|---|---|
| `#/splash` | S0 | 없음(항상 최초) |
| `#/signup` | S1 | 비로그인 or 미인증 |
| `#/map` | S2 | **로그인 + 거주 인증** 필수 |
| `#/venue/:id` | S3/S4 | 로그인 + `can_view_region(매장 지역)` |
| `#/host-apply/:venueId` | S5 | 로그인 + **홈 지역**(방문자 차단) |
| `#/joined` | S6 | 로그인 |
| `#/my` | S7a | 로그인 + 인증 |
| `#/passes` | S8 | 로그인 + 인증 |
| `#/me` | S7b | 로그인 + 인증 |

**가드 규칙**
- 미로그인 → `#/signup`. 로그인·미인증 → `#/signup`의 거주 인증 단계.
- `map`/상세는 **거주 인증** 없으면 진입 불가(빈 지도 대신 게이트).
- 상세(`venue/:id`)는 **볼 수 있는 지역**(홈 또는 이용권)만. 아니면 이용권 유도(→`#/passes`).
- `host-apply`는 **홈 지역 매장만**. 방문자·타지역 → 차단 토스트.
- 매장 상세는 매장 상태에 따라 S3(진행중=주황) / S4(공간만=어두움)로 분기 렌더.

## 5. 매장 라이프사이클 — 핀 상태 머신 ★

지도의 핀 색은 곧 매장의 상태다.

```
[운영진 협의·등록]
      │  venues insert (시설 + 프로그램 후보 + 유휴시간)
      ▼
● 어두운 핀  (open_space: 공간만, 프로그램 없음)
      │  호스트가 S4→S5에서 프로그램 기획 + 대여 신청
      ▼
◐ 대여 신청됨 (pending: 승인 대기)   ← MVP 옵션 A: 즉시 오픈이면 이 단계 생략
      │  (운영진/매장 승인 or 자동)
      ▼
🟠 주황 네온 핀 (live: 프로그램 진행중, 모집중)
      │  정원 마감 → 여전히 live(모집 마감 뱃지)
      │  프로그램 종료(starts_at + duration 경과)
      ▼
● 다시 어두운 핀 (공간 재개방) 또는 done 아카이브
```

- **한 매장에 동시 프로그램 수**: MVP는 **유휴 슬롯당 1개**(단순화). 여러 유휴 슬롯이면 슬롯별 상태.
- **데이터 매핑**: 어두운 핀 = 해당 지역 `venues` 중 활성 `meetups(status='open')` 없음. 주황 핀 = 활성 `meetups` 있음. (지도 쿼리는 매장+활성모임 조인.)
- **승인 흐름**: MVP 권장 = **즉시 오픈**(대여신청=모임개설, admin은 사후 모니터). 승인제(pending)는 백로그. → 상세는 `05` S5.

## 6. 전역 상태 (app state)

| 키 | 의미 |
|---|---|
| `session` | Supabase 세션(로그인 여부) |
| `profile` | 닉네임·관심사·홈 `region_id`·`resident_verified`·`is_host` |

| `mode` | live(Supabase) / mock(체험) — `IS_MOCK` |

## 7. 사용자 여정 (골든 패스)

**참여자**
`S0 → S1(가입·관심사·거주인증) → (열림 연출) → S2 지도 → 🟠주황 핀 → S3 매장상세 → 참여 신청 → S6 완료`

**호스트**
`S2 지도 → ●어두운 핀 → S4 매장상세(시설·프로그램 후보) → S5 기획·대여 신청 → (매칭) → 그 핀이 🟠주황으로`

**방문자(크로스-리전)**
`S2 지도 → 지역 ▾ / ＋다른 지역 → S8 이용권 발급 → viewRegion 전환 → 🟠핀 → S3 → 참여`

**운영진(앱 밖·후순위)**
`가게 섭외·협의 → A0 매장 등록(시설·프로그램 후보·유휴시간) → 지도에 ●어두운 핀 등장`
