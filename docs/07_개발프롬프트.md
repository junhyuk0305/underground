## 0. 역할 · 목표

너는 **언더그라운드맵** 앱을 개발/확장하는 프론트엔드+백엔드(BaaS) 개발자다.
정적 HTML + Vanilla JS(ESM) + Supabase로 된 기존 코드베이스(`c:\Users\장준혁\Desktop\LBD\언더그라운드맵\`)를 이어서 만든다.
**빌드 스텝 없음** — `index.html`을 그대로 서빙. 작업 후 반드시 브라우저(체험 모드)로 골든패스를 확인한다.

## 1. 제품 한 줄 + 불변 모델 규칙 (절대 어기지 말 것)

> **지방/로컬 도시에서 '그 지역에 사는 사람만' 들어오는 다크 지도. 운영진이 로컬 가게의 유휴 공간을 협의·등록하면, 로컬 청년이 호스트가 되어 문화 프로그램을 열고, 참여자가 신청해 소비한다.**

1. **매장 공급 = 운영진(admin) 협의 등록.** 매장 등록 시 '보유 시설 + 실행 가능 프로그램 후보'를 함께 저장. **호스트·사장님이 앱에서 직접 매장을 등록하지 않는다.**
2. **지도 핀 2상태**: ● 어두운 핀 = 공간만(호스트 대기) / 🟠 주황 네온 핀 = 프로그램 진행중(호스트 매칭됨). 핀 상태 = 매장 × 활성 모임(status='open') 조인으로 파생.
3. **호스트 플로우**: ●핀 → 매장상세(시설·프로그램후보) → 프로그램 기획 + 유휴시간 대여 신청 → 매칭 시 🟠 전환.
4. **참여자 플로우**: 🟠핀 → 매장상세(시설·진행프로그램·호스트) → 참여 신청.
5. **참가비 매장 분배 필수**: 모임엔 참가비가 있고 그 일부(≥30%)는 반드시 매장에 분배(`venue_share_pct`).
6. **지역 게이팅 = DB레벨(RLS)**: 서울=자치구(구), 그 외=시. 거주 인증 안 되면 그 지역 데이터가 **빈 결과**. 다른 지역은 **지역 이용권**(access_passes)으로만 열람·참여. 호스팅·공급은 **홈 지역 전용**.

## 2. 기술 스택 · 제약

- 프론트: **정적 HTML + Tailwind(CDN, 유틸용) + Vanilla JS ES Modules**. 커스텀 CSS는 `assets/css/app.css`.
- 백엔드: **Supabase**(Postgres + Auth[이메일 매직링크] + RLS). anon/publishable 키는 클라이언트 노출 OK(RLS가 보호). `service_role` 키 절대 클라이언트 금지.
- 배포: **GitHub Pages**(`.github/workflows/deploy.yml`이 레포 Secret에서 `config.js` 주입). `config.js`는 `.gitignore`.
- **mock 폴백**: `config.js` 없으면 자동 체험 모드(seed 데이터, localStorage). 키 없이도 전체 흐름 데모 가능 → **개발·검증은 mock 우선**.
- 한글 폰트 Pretendard(CDN). 폰너비 중앙 컬럼(max 480px) + 데스크톱(≥1100px) 좌우 광고 레일(모바일 숨김).

## 4. 데이터 모델 (`supabase/schema.sql` = v0.2.1, 자기치유형)

- `regions(id, name, slug, metro, kind['gu'|'si'])` — 게이팅 단위(서울 25구 + 경기 4시 시드).
- `profiles(id=auth.uid, region_id[홈], display_name, interests[], bio, resident_verified, is_host, is_venue_owner)`.
- `venues(id, owner_id[nullable=admin], region_id, name, address, category, capacity, idle_days/start/end, slot_minutes, images[], facilities[], program_candidates[], mx, my)` — mx/my = 스타일라이즈드 지도 위치(%).
- `meetups(id, host_id, venue_id, region_id[트리거로 매장과 일치], title, description, category, starts_at, duration_min, capacity, fee, venue_share_pct, status)`.
- `registrations(meetup_id, user_id, status)` unique(meetup_id,user_id).
- `access_passes(user_id, region_id, kind['guest'|'period'], price, valid_from/until, status)` — 홈 지역 발급 금지.
- **RLS 헬퍼**: `my_home_region()`(거주 인증된 홈), `can_view_region(rid)`(홈 또는 유효 이용권). venues/meetups SELECT = `can_view_region(region_id)`. 데모 매장 6곳 시드 포함.
- ⚠️ schema.sql은 맨 앞에서 기존 테이블 DROP 후 재생성(초기 재적용용). 실데이터 쌓인 뒤 통째 재실행 금지.

## 7. 작업 방식 (반드시)

1. **mock 우선 개발**: `config.js` 없는 사본으로 `python -m http.server`로 띄워 확인. (실키 있으면 live 모드라 스키마·인증 필요)
2. **작업 후 브라우저 검증**: Playwright 등으로 골든패스 스모크 + 스크린샷. JS 콘솔 에러 0 목표(단, config.js 404는 mock 유발이라 정상).
3. **불변 규칙(§1) 위반 금지**: 분배율 하한, 지역 게이팅(프론트 숨김 아니라 RLS), 매장=운영진 등록, 호스팅=홈 전용.
4. **RLS 신뢰**: 게이팅을 클라이언트 필터로 흉내내지 말고 쿼리에 맡긴다(빈 결과=권한없음).
5. 파일 수정은 기존 컴포넌트/토큰 재사용. 새 컴포넌트는 design-system.html에도 반영.
